# 酒馆助手模板 — 开发操作笔记

> 写给下一个接手这个项目的 AI / 开发者。本项目是 SillyTavern 的辅助脚本/前端界面模板，通过 iframe 注入到 SillyTavern 中使用。

---

## 1. 项目结构速览

```
tavern_helper_template-main/
├── src/                          # 源码目录（改这里）
│   ├── Zsd网游论坛/               # 网游论坛模块源码
│   │   ├── index.ts              # 入口文件（iframe 创建、Vue 挂载、移动端适配）
│   │   ├── App.vue               # Vue 主组件
│   │   └── ...
│   └── 悬浮球/                    # 其他模块...
├── dist/                         # 编译输出（由 webpack 自动生成）
│   └── Zsd网游论坛/
│       └── index.js              # SillyTavern 实际加载的脚本
├── util/                         # 工具库（script.ts, mvu.ts 等）
├── 初始模板/                      # 空白模板参考
├── 示例/                          # 示例代码参考
├── cors_server.py                # 本地 CORS HTTP 服务器（见下方）
└── package.json                  # pnpm 脚本配置
```

**关键规则**：
- 改 `src/` 下的源码 → 热编译自动更新 `dist/` → SillyTavern 加载 `dist/` 中的 JS
- `dist/` 已加入 git，但 `.gitattributes` 中设置了 `dist/** merge=ours`（合并时保留本地 dist）

## 2. 热编译（Hot Reload）

### 启动监听
```powershell
pnpm watch
```

- 这会启动 **webpack watch 模式** + **WebSocket 实时推送**
- 修改 `src/` 下的任何 `.ts` / `.vue` 文件后，**自动编译**到 `dist/`
- WebSocket 端口默认 **6621**，用于通知浏览器刷新（但 SillyTavern 内嵌 iframe 通常需要手动刷新页面）

### 验证修改是否生效
1. 保存源码文件
2. 等待控制台出现 `[webpack-cli] Compilation finished` 或类似提示
3. 在 SillyTavern 中 **刷新页面**（F5 或重新加载）
4. 如使用 DevTools：在移动设备模拟模式（iPhone 375×812）下验证

> **注意**：浏览器可能缓存 JS。如需强制刷新，在 SillyTavern 地址栏加 `?t=1` 或 DevTools Network 面板勾选 Disable Cache。

## 3. 本地开发服务器（CORS）

SillyTavern 通过 `http://localhost:5500/Zsd网游论坛/index.js` 加载脚本，需要 CORS 头。

### 启动服务器
```powershell
python cors_server.py
```

- 端口：**5500**
- 自动添加 `Access-Control-Allow-Origin: *` 头
- 如果端口被占，修改 `cors_server.py` 中的 `PORT` 变量，并同步修改 SillyTavern 中的脚本 URL

### 背景任务管理
如需要后台长期运行，可用 PowerShell：
```powershell
# 启动 webpack watch（后台）
Start-Process powershell -ArgumentList "-Command pnpm watch"
# 启动 CORS 服务器（后台）
Start-Process powershell -ArgumentList "-Command python cors_server.py"
```

## 4. 移动端调试技巧

### Chrome DevTools MCP 用法
1. 用 `emulate` 工具设置移动视口：`375x812x3,mobile,touch`
2. 用 `take_snapshot` 查看 a11y 树结构
3. 用 `evaluate_script` 执行 DOM 检查
4. 用 `click` 或 `evaluate_script` 模拟按钮点击

### 常见问题：弹窗/元素看不到
- 用 `evaluate_script` 检查 `getBoundingClientRect()`
- 常见问题：`bottom: 0` + `height: 85vh` 会导致元素顶部溢出视口（`top` 为负值）
- 修复方式：改用 `top: 10vh`（从视口顶部开始定位）

## 5. 提交代码到 GitHub

```powershell
# 查看变更
git status

# 添加关键文件（通常只需 src/ 和对应的 dist/）
git add "src/Zsd网游论坛/index.ts" "dist/Zsd网游论坛/index.js"

# 提交
git commit -m "fix(mobile): xxx 修复描述"

# 推送
git push origin main
```

### 注意事项
- 每次修改源码后，**必须同时提交 `dist/`** 编译输出，否则 SillyTavern 加载的还是旧代码
- 如果 `dist/` 有额外附带变更（如悬浮球、示例目录），确认是否为本轮修改产生，避免混入无关内容
- commit message 建议用 `fix(...):` / `feat(...):` / `refactor(...):` 前缀

## 6. SillyTavern 中加载脚本的 URL

在 SillyTavern 的 **JS 扩展 / Slash Runner** 中，通过以下 URL 加载：

```
http://localhost:5500/Zsd网游论坛/index.js
```

如果服务器地址或端口变更，同步修改此 URL。

## 7. 快速参考：本次修复记录

| 问题 | 原因 | 修复 |
|------|------|------|
| 移动端论坛弹窗看不到 | `bottom: 0` + `height: 85vh` 导致顶部溢出视口 | 改为 `top: '10vh'` |

| CI bundle 推送失败 | tag 排序用了默认字典序 | 改为 `sort -V` 版本号排序 |
