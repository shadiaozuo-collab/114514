# 星·数据库 v2.6 核心方法蒸馏

> 原始脚本: `星数据库v2.6-原始脚本.js`（Tampermonkey 用户脚本，约 1.5MB）
>
> 本文档提取其**AI 指令预设系统、API 配置与调用、多层内部存储**三大核心机制，并适配为酒馆助手（JS-Slash-Runner）环境下的参考实现。
>
> **关键认知修正**：原脚本的核心价值不在于 API 连接配置，而在于其**完整的 AI 指令预设（messages 数组）管理系统**——包括破甲前缀、人格设定、任务指令、背景数据注入、A/B 槽位动态替换等。

## 一、AI 指令预设系统（核心）

### 1.1 设计思想

原脚本将发给 AI 的完整提示词组织为一个 **messages 数组**（类似 OpenAI Chat API 格式），每个元素包含：

```typescript
interface PromptSegment {
  role: 'SYSTEM' | 'USER' | 'assistant';
  content: string;       // 提示词内容（含占位符如 $0, $1, $U 等）
  deletable: boolean;    // 是否允许用户在 UI 中删除
  mainSlot?: 'A' | 'B';  // A/B 主提示词槽位标记
  isMain?: boolean;      // 兼容旧版的 A 槽标记
  isMain2?: boolean;     // 兼容旧版的 B 槽标记
}
```

脚本内置三套独立的 AI 指令预设：

| 预设名称 | 用途 | 核心特征 |
|---------|------|---------|
| `charCardPrompt` | **填表功能** | 默认 10 段 messages，含破甲 + 人格设定 + 背景注入 + 填表指令 |
| `plotSettings.promptGroup` | **剧情推进** | 独立 messages 数组，recall + supplement 双系统工作流 |
| `contentOptimizationSettings.promptGroup` | **正文优化** | 输出 JSON 格式优化建议 |

### 1.3 占位符替换系统

在发请求前，脚本会遍历所有 `PromptSegment.content`，替换以下占位符：

| 占位符 | 含义 | 数据来源 |
|--------|------|---------|
| `$0` | 当前表格数据 | 动态生成 |
| `$1` | 正文/历史消息文本 | 酒馆聊天记录 |
| `$4` | 世界书内容 | 酒馆世界书 |
| `$5` | 总结大纲 | 剧情推进生成 |
| `$6` | 上轮剧情规划内容 | 剧情推进缓存 |
| `$7` | 前文上下文 | 上下文提取 |
| `$8` | 额外提示 / 用户本轮输入 | 手动输入或剧情推进注入 |
| `$U` | 用户基础设定 | 酒馆用户设定 |
| `$C` | 角色描述 | 酒馆角色卡 |

替换代码示例：

```typescript
function renderPromptSegments(
  segments: PromptSegment[],
  placeholders: Record<string, string>
): Array<{ role: string; content: string }> {
  return segments.map(seg => {
    let content = seg.content;
    // 基础占位符替换
    content = content.replace(/\$0/g, placeholders.tableData || '');
    content = content.replace(/\$1/g, placeholders.messagesText || '');
    content = content.replace(/\$4/g, placeholders.worldbookContent || '');
    content = content.replace(/\$5/g, placeholders.summaryOutline || '');
    content = content.replace(/\$6/g, placeholders.lastPlotContent || '');
    content = content.replace(/\$7/g, placeholders.contextText || '');
    content = content.replace(/\$8/g, placeholders.manualHint || '');
    content = content.replace(/\$U/g, placeholders.userInfo || '');
    content = content.replace(/\$C/g, placeholders.charInfo || '');
    // 高级模板解析（可选）
    content = parseRandomTags(content);      // {{random:选项1|选项2}}
    content = parseCalcTags(content);        // {{calc:表达式}}
    content = parseIfBlocks(content);        // {{if:条件}}...{{endif}}
    return { role: seg.role.toLowerCase(), content };
  }).filter(m => m.content.trim().length > 0);
}
```

### 1.4 A/B 槽位动态替换机制（核心创新）

这是原脚本最重要的设计：**剧情推进功能可以临时"劫持"填表提示词**。

**原理**：填表提示词中标记了 `mainSlot="A"` 和 `mainSlot="B"` 的两段是不可删除的"主提示词槽位"。当剧情推进触发时，脚本会将这两段的内容临时替换为剧情推进专用的提示词：

```typescript
let plotOverrideBackup: PromptSegment[] | null = null;
let plotOverrideActive = false;

/**
 * 应用剧情推进提示词覆盖
 * 将 charCardPrompt 中 A/B 槽位临时替换为剧情推进提示词
 */
function applyPlotPromptOverride(settings: ScriptSettings): void {
  if (plotOverrideActive || !settings.plotSettings?.enabled) return;

  const plotMain = settings.plotSettings.prompts.find(p => p.id === 'mainPrompt')?.content?.trim() || '';
  const plotTask = settings.plotSettings.prompts.find(p => p.id === 'systemPrompt')?.content?.trim() || '';
  if (!plotMain && !plotTask) return;

  // 备份原始填表提示词
  plotOverrideBackup = JSON.parse(JSON.stringify(settings.charCardPrompt));

  // 在副本中找到 A/B 槽位并替换内容
  const getSlot = (seg: PromptSegment) =>
    seg.mainSlot?.toUpperCase() || (seg.isMain ? 'A' : seg.isMain2 ? 'B' : '');

  const modified = JSON.parse(JSON.stringify(settings.charCardPrompt));
  const aIdx = modified.findIndex((p: PromptSegment) => getSlot(p) === 'A');
  const bIdx = modified.findIndex((p: PromptSegment) => getSlot(p) === 'B');

  if (plotMain && aIdx !== -1) modified[aIdx].content = plotMain;
  if (plotTask && bIdx !== -1) modified[bIdx].content = plotTask;

  settings.charCardPrompt = modified;
  plotOverrideActive = true;
}

/**
 * 恢复原始填表提示词
 */
function restorePlotPromptOverride(settings: ScriptSettings): void {
  if (!plotOverrideActive || !plotOverrideBackup) return;
  settings.charCardPrompt = plotOverrideBackup;
  plotOverrideBackup = null;
  plotOverrideActive = false;
}
```

**使用时机**：

```typescript
// 剧情推进开始前
applyPlotPromptOverride(settings);

// 调用填表 API（此时实际使用的是被剧情提示词"劫持"的版本）
const result = await callCustomOpenAI(settings.charCardPrompt, placeholders);

// 剧情推进结束后恢复
restorePlotPromptOverride(settings);
```

## 二、API 配置与调用机制

### 2.1 三种 API 调用模式

| 模式 | 说明 |
|------|------|
| `custom` | 使用用户填写的自定义 API URL + Key + Model |
| `tavern` | 使用酒馆的连接预设（Connection Profile）|
| `useMainApi` | 直接复用酒馆当前正在使用的 API 和模型 |

在**酒馆助手环境**下，`generate` / `generateRaw` 已原生支持 `custom_api` 参数。

### 2.2 调用入口（适配酒馆助手）

```typescript
async function callAI(
  userInput: string,
  settings: ScriptSettings,
  options?: {
    should_stream?: boolean;
    should_silence?: boolean;
    overrides?: Overrides;
    injects?: Omit<InjectionPrompt, 'id'>[];
    ordered_prompts?: (BuiltinPrompt | RolePrompt)[];
  }
): Promise<string> {
  const { apiMode, apiConfig, tavernProfile } = settings;

  if (apiConfig.useMainApi) {
    return generate({ user_input: userInput, ...options });
  }

  if (apiMode === 'tavern' && tavernProfile) {
    return generate({
      user_input: userInput,
      custom_api: { proxy_preset: tavernProfile, model: apiConfig.model || undefined },
      ...options,
    });
  }

  return generate({
    user_input: userInput,
    custom_api: {
      apiurl: apiConfig.url, key: apiConfig.apiKey, model: apiConfig.model, source: 'openai',
      max_tokens: apiConfig.max_tokens, temperature: apiConfig.temperature,
    },
    ...options,
  });
}
```

## 三、多层存储回退机制

### 3.1 核心设计思想

原脚本实现了**三层存储回退**策略：

```
读取优先级：酒馆设置 (extension_settings) → IndexedDB → localStorage
写入策略：  酒馆设置 + IndexedDB 同时写入
```

在**酒馆助手环境**下，推荐简化为两层：

```
读取优先级：酒馆变量 (getVariables) → IndexedDB
写入策略：  酒馆变量 + IndexedDB 同时写入
```

> **为何不使用 localStorage？**
> 原脚本明确禁用了 localStorage（`FORBID_BROWSER_LOCAL_STORAGE_FOR_CONFIG_ACU = true`），因为 localStorage 在 iframe 跨域场景下不可用，且数据无法跨浏览器同步。

### 3.2 统一存储接口实现

```typescript
const SCRIPT_ID = 'my_script_v1';
const IDB_DB_NAME = `${SCRIPT_ID}_config`;
const IDB_STORE_NAME = 'kv';

// -- IndexedDB 底层 --

let idbDbPromise: Promise<IDBDatabase | null> | null = null;

function openIdbDb(): Promise<IDBDatabase | null> {
  if (idbDbPromise) return idbDbPromise;
  idbDbPromise = new Promise((resolve) => {
    try {
      const req = indexedDB.open(IDB_DB_NAME, 1);
      req.onupgradeneeded = () => {
        const db = req.result;
        if (!db.objectStoreNames.contains(IDB_STORE_NAME)) {
          db.createObjectStore(IDB_STORE_NAME);
        }
      };
      req.onsuccess = () => resolve(req.result);
      req.onerror = () => {
        console.warn('[Storage] IndexedDB open failed');
        resolve(null);
      };
    } catch (e) {
      resolve(null);
    }
  });
  return idbDbPromise;
}

async function idbSet(key: string, value: string): Promise<void> {
  const db = await openIdbDb();
  if (!db) return;
  const tx = db.transaction(IDB_STORE_NAME, 'readwrite');
  const store = tx.objectStore(IDB_STORE_NAME);
  store.put(value, key);
  return new Promise((resolve, reject) => {
    tx.oncomplete = () => resolve();
    tx.onerror = () => reject(tx.error);
  });
}

async function idbGet(key: string): Promise<string | null> {
  const db = await openIdbDb();
  if (!db) return null;
  const tx = db.transaction(IDB_STORE_NAME, 'readonly');
  const store = tx.objectStore(IDB_STORE_NAME);
  const req = store.get(key);
  return new Promise((resolve) => {
    req.onsuccess = () => resolve(req.result ?? null);
    req.onerror = () => resolve(null);
  });
}

async function idbRemove(key: string): Promise<void> {
  const db = await openIdbDb();
  if (!db) return;
  const tx = db.transaction(IDB_STORE_NAME, 'readwrite');
  const store = tx.objectStore(IDB_STORE_NAME);
  store.delete(key);
}

// -- 统一存储接口 --

interface StorageBackend {
  getItem(key: string): string | null;
  setItem(key: string, value: string): void;
  removeItem(key: string): void;
}

/**
 * 获取统一存储接口
 *
 * 读取优先级：酒馆变量 > IndexedDB
 * 写入策略：酒馆变量 + IndexedDB 同时写入
 */
function getUnifiedStorage(scriptId: string): StorageBackend {
  // 酒馆变量命名空间
  const varsKey = `${scriptId}_settings`;

  return {
    getItem(key: string): string | null {
      // 优先级 1: 酒馆变量（跨浏览器同步）
      try {
        const vars = getVariables({ type: 'script', script_id: scriptId });
        if (vars && typeof vars[key] === 'string') {
          return vars[key];
        }
      } catch (e) {
        // 酒馆变量不可用
      }

      // 优先级 2: IndexedDB（仅本浏览器）
      // 注意：IndexedDB 是异步的，这里返回同步结果需要缓存策略
      // 简化方案：在脚本启动时预加载 IndexedDB 到内存缓存
      const cached = idbMemoryCache.get(key);
      if (cached !== undefined) return cached;

      return null;
    },

    setItem(key: string, value: string): void {
      // 写入酒馆变量
      try {
        const vars = getVariables({ type: 'script', script_id: scriptId }) || {};
        vars[key] = value;
        insertOrAssignVariables(vars, { type: 'script', script_id: scriptId });
      } catch (e) {
        console.warn('[Storage] Failed to write to tavern variables:', e);
      }

      // 同时写入 IndexedDB
      idbMemoryCache.set(key, value);
      void idbSet(key, value);
    },

    removeItem(key: string): void {
      try {
        const vars = getVariables({ type: 'script', script_id: scriptId }) || {};
        delete vars[key];
        insertOrAssignVariables(vars, { type: 'script', script_id: scriptId });
      } catch (e) {
        // ignore
      }
      idbMemoryCache.delete(key);
      void idbRemove(key);
    },
  };
}

// 内存缓存（解决 IndexedDB 异步读取问题）
const idbMemoryCache = new Map<string, string>();

/** 脚本启动时预加载 IndexedDB 到内存缓存 */
async function preloadIdbCache(scriptId: string): Promise<void> {
  const db = await openIdbDb();
  if (!db) return;
  const tx = db.transaction(IDB_STORE_NAME, 'readonly');
  const store = tx.objectStore(IDB_STORE_NAME);
  const req = store.openCursor();
  req.onsuccess = () => {
    const cursor = req.result;
    if (cursor) {
      idbMemoryCache.set(cursor.key as string, cursor.value as string);
      cursor.continue();
    }
  };
}
```

### 4.2 独立浮动窗口（jQuery 实现）

原脚本完全自行实现了一个不依赖酒馆 `callGenericPopup` 的窗口系统，核心特征：

- **挂载到顶层窗口**：解决 iframe 嵌套问题
- **支持拖拽、缩放、最大化**
- **记住窗口位置和尺寸**
- **双主题切换**

在酒馆助手中，由于可以直接使用 Vue + jQuery，推荐方案：

import DatabaseUI from './DatabaseUI.vue';

### D. 字数阈值检测（避免 AI 截断/空回误触发）

星数据库在**自动填表**和**剧情推进**两个入口都加了"AI 回复过短则跳过"的防护，避免用截断/空回内容生成无效数据。

#### 自动填表层（最常用）

核心思路：**取最后一条 AI 消息的 `.mes` 字符长度，与阈值比较**。

```typescript
// 阈值配置（默认 500 字符）
const DEFAULT_AUTO_UPDATE_TOKEN_THRESHOLD = 500;

function shouldSkipDueToShortReply(
  chatHistory: any[],
  threshold: number,
  isAutoUpdateMode: boolean,
): boolean {
  if (!isAutoUpdateMode || threshold <= 0) return false;

  // 找到当前批次中最后一条 AI 消息
  const lastAiMessageInBatch = chatHistory[lastMessageIndexOfBatch];
  const content = lastAiMessageInBatch?.mes || lastAiMessageInBatch?.message || '';
  const length = content.length;

  if (length < threshold) {
    console.log(`[Auto] 跳过：AI 回复长度 ${length} 低于阈值 ${threshold}`);
    return true;
  }
  return false;
}
```

**要点**：
- 检测对象是**最后一条 AI 消息**，不是整个上下文
- `length` 是 JavaScript 字符串长度（字符数），不是 Token 数
- 仅对 `auto_*` 模式生效，手动更新强制跳过检测
- 默认阈值 **500**，UI 中可配置

### E. 对话楼层回退同步机制

当用户 **Swipe** 或 **删除消息** 时，星数据库能让已生成的表格/剧情数据"跟着回退"。

#### 核心设计：数据嵌入 AI 消息

不存全局变量，而是把每轮生成的数据**写入对应 AI 消息的 extra 字段**：

```typescript
// 每条 AI 消息的数据结构
interface ChatMessage {
  mes: string;
  name: string;
  is_user: boolean;
  extra?: {
    TavernDB_ACU_IsolatedData?: {
      [isolationKey: string]: {
        independentData: {
          [sheetKey: string]: any; // 该轮生成的表格数据
        };
      };
    };
  };
}
```

#### 回退触发：监听 MESSAGE_DELETED / MESSAGE_SWIPED

```typescript
const chatModificationEvents = ['MESSAGE_DELETED', 'MESSAGE_SWIPED'];

chatModificationEvents.forEach(evName => {
  eventSource.on(eventTypes[evName], async () => {
    // 防抖后重新全量合并
    clearTimeout(debounceTimer);
    debounceTimer = setTimeout(async () => {
      await refreshMergedDataAndNotify();
    }, 500);
  });
});
```

#### 合并算法：从后往前扫描，取"每个表最新"的数据

```typescript
function mergeAllIndependentTables(chat: ChatMessage[]) {
  const mergedData = {};
  const foundSheets = {}; // 记录已找到的表，避免覆盖

  // 【核心】从最后一条消息往前遍历
  for (let i = chat.length - 1; i >= 0; i--) {
    const message = chat[i];
    if (message.is_user) continue; // 跳过用户消息

    const tagData = message.extra?.TavernDB_ACU_IsolatedData?.[currentIsolationKey];
    const independentData = tagData?.independentData || {};

    Object.keys(independentData).forEach(sheetKey => {
      // 只处理模板中存在的表
      if (!templateSheetKeySet.has(sheetKey)) return;

      // 【关键】如果这张表已经在更晚的消息中找到过，就跳过
      // 因为是从后往前遍历，第一次遇到 = 最新消息中的数据
      if (!foundSheets[sheetKey]) {
        mergedData[sheetKey] = JSON.parse(JSON.stringify(independentData[sheetKey]));
        foundSheets[sheetKey] = true;
      }
    });
  }

  return mergedData;
}
```

#### 为什么这样能"回退"？

| 操作 | 聊天记录变化 | 重新扫描结果 |
|------|------------|------------|
| 正常推进 | `chat` 增加一条 AI 消息 | 从新消息中取到最新数据 |
| Swipe 替换 | 最后一条 AI 消息被替换 | 新消息中可能没有该表数据，会往前找到旧数据 |
| 删除消息 | `chat.length` 变短 | 遍历范围自然缩短，取到的就是更早状态 |

**本质**：没有显式的"快照/undo 栈"，回退是**被动回溯**——数据分散在每一层 AI 消息中，删除后重新从尾部向前扫描，取到的自然就是"剩余楼层中最新"的状态。
