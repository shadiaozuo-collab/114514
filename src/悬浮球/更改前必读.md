# 更改前必读

## 技术栈约定

**悬浮球本体请尽量以 jQuery 为主。**

当前脚本的悬浮球部分（DOM 创建、事件绑定、拖拽逻辑、样式注入）全部使用纯 jQuery 实现。虽然项目中可以引入 Vue，但：

- Vue 目前只用于**设置界面**（挂载在 `#extensions_settings2` 的那个折叠面板）
- 悬浮球的按钮、面板、交互逻辑都是 jQuery 写的
- 如果后面 AI token 一多，很容易遗忘这一点，结果在 jQuery 的代码里硬塞 Vue 的写法，反而搞出一堆 bug

所以：悬浮球相关的 UI 变更，优先用 jQuery 搞定；只有在设置面板这种需要双向数据绑定的地方，才用 Vue + Pinia。

## 文件分工

| 文件 | 职责 | 技术 |
|------|------|------|
| `index.ts` | 悬浮球主逻辑 | jQuery |
| `settings.ts` | 设置数据的 zod 校验 + pinia store | Pinia |
| `设置界面.vue` | 酒馆设置页的折叠面板 | Vue |

## 提醒事项

1. **样式是内联字符串**：悬浮球的 CSS 写在 `index.ts` 的 `styles` 模板字符串里，改样式直接去那里搜 `.fb-xxx`，不要新建 CSS 文件。

2. **设置默认值在 `settings.ts`**：如果要给悬浮球加新的可配置项，先在 `settings.ts` 的 zod schema 里定义字段和默认值，再去 `设置界面.vue` 里加输入框，最后到 `index.ts` 里用 `settings.xxx` 读取。

3. **拖拽和展开状态用 `localStorage` 存，设置项用脚本变量存**：不要混。位置和收起状态存在浏览器本地；图标、标题等存在酒馆脚本变量里（由 `settings.ts` 自动同步）。

4. **卸载清理**：`pagehide` 事件里已经写了移除 DOM 和解除事件绑定，如果后面新增了什么全局事件或定时器，记得也加到清理逻辑里。

5. **不要碰 `dist/` 目录**：那是 webpack 打包输出，改 `src/` 里的源文件，然后运行 `pnpm dev` 或 `pnpm build` 自动生成。

6. **构建前确保依赖已安装**：如果 `pnpm build` 报错说 `webpack` 找不到或 `node_modules missing`，先运行 `pnpm install`。这个项目没有自带 node_modules。

7. **脚本在酒馆助手里加载，不用找酒馆安装目录**：在酒馆助手新建脚本，内容写 `import 'http://localhost:5500/dist/悬浮球/index.js'`（端口根据你启动的 HTTP 服务调整）。参考 `初始模板/脚本/导入到酒馆中/脚本-实时修改.json`。`dist` 里的中文文件夹名在命令行下可能显示乱码，不用管，看 `LastWriteTime` 确认是最新生成的就行。

8. **如何打包上传（改完代码后必做）**：
   - **第 1 步：打包** — 在项目根目录运行 `pnpm build`（或 `npx webpack --mode production`），构建产物输出到 `dist/悬浮球/index.js`。
   - **第 2 步：启动 HTTP 服务** — 确保有 HTTP 服务器在提供 `dist/` 目录，例如运行 `python -m http.server 5500 --directory dist`（也可用 VS Code Live Server 等工具，注意根目录要指向 `dist/` 而不是项目根目录）。
   - **第 3 步：酒馆中加载** — 在酒馆助手的脚本库中新建或编辑脚本，内容为 `import 'http://localhost:5500/悬浮球/index.js'`，保存后启用即可。
   - **第 4 步：刷新验证** — 如果脚本已加载但没有更新，在酒馆中禁用再重新启用脚本，或刷新酒馆页面。看悬浮球面板内容是否已变为最新修改。
