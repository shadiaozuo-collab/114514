# 网游论坛悬浮窗 - 实现计划

## 概述

在酒馆中创建一个网游风格的论坛悬浮窗，以**脚本项目**形式实现。使用 `createScriptIdIframe` 创建独立 iframe 挂载到酒馆页面，支持自由拖动。论坛分为「现实讨论」和「游戏中论坛」两大板块，帖子和评论由 AI 根据可自定义的提示词生成，使用酒馆当前 API 源 + 自定义模型。

## 数据结构

```typescript
type ForumSection = '现实讨论' | '游戏中论坛';

interface ForumPost {
  id: string;

  content: string;

  id: string;
  authorId: string;
  content: string;
  timestamp: number;
  isAiGenerated: boolean;
}

interface ForumSettings {
  forumName: string;
  model: string;           // 自定义模型名
  prompt_现实讨论: string;   // 现实讨论板块的系统提示词
  prompt_游戏中论坛: string;  // 游戏中论坛板块的系统提示词

```

## 双向上下文注入（核心特性）

论坛与酒馆对话之间存在双向上下文流动：

### 论坛 → 酒馆对话（注入提示词）

- 使用 `injectPrompts` 将论坛帖子内容作为系统提示词注入到酒馆 AI 请求中
- 注入格式：将当前板块的帖子摘要（标题 + 作者 + 内容 + 部分评论）格式化为文本
- 注入位置：`position: 'in_chat'`, `depth: 0`, `role: 'system'`
- 每次论坛帖子/评论更新时，重新格式化注入内容
- 监听 `tavern_events.GENERATION_AFTER_COMMANDS` 事件，在每次 AI 生成前确保注入内容是最新的
- 监听 `tavern_events.CHAT_CHANGED` 事件，在新聊天文件中重新注入
- 用户可在设置中选择是否开启此功能，以及注入的帖子数量上限

### 酒馆对话 → 论坛（AI 生成上下文）

- 生成论坛帖子/评论时，使用 `getChatMessages` 获取最近的对话历史
- 将对话历史摘要作为上下文传给 `generateRaw` 的 `ordered_prompts`
- 论坛 AI 会根据当前剧情走向生成相关帖子和评论
- 例如：剧情中发生了重大事件 → 游戏中论坛出现相关讨论帖
- 用户可在设置中配置注入的聊天历史条数（默认最近 10 条）

## 核心功能模块

### 1. 悬浮窗与拖动 (`index.ts` + `App.vue`)

- 脚本项目，使用 `createScriptIdIframe()` 创建独立 iframe
- 使用 `jqueryui` 的 draggable 实现拖动
- 悬浮窗初始位置在酒馆页面右侧，可拖到任意位置
- 支持最小化（收起为小图标）和关闭
- 通过 `appendInexistentScriptButtons` 注册按钮，点击切换悬浮窗显隐
- 使用 `teleportStyle($app[0].contentDocument!.head)` 复制样式到 iframe

### 2. 设置持久化 (`settingsStore.ts`)

- 使用 zod 定义 `ForumSettings` 的 schema 和默认值
- 使用 `getVariables` / `replaceVariables` 配合 `{ type: 'script', script_id: getScriptId() }` 持久化到脚本变量
- 使用 pinia + `watchEffect` 实现响应式读写
- 默认值示例：
  - `forumName`: "虚拟网游论坛"
  - `model`: "" (空字符串表示使用酒馆当前模型)
  - `prompt_现实讨论`: "你是一个现实世界的论坛用户..."
  - `prompt_游戏中论坛`: "你是一个网游中的玩家..."
  - `enableInjectToChat`: true
  - `injectPostCount`: 5
  - `injectChatHistoryCount`: 10

### 3. AI 帖子/评论生成 (`aiGenerator.ts`)

- 使用 `generateRaw` 调用 AI，配置 `custom_api` 仅覆盖 `model` 字段
- 使用 `ordered_prompts` 自定义提示词顺序，不携带酒馆预设
- **对话上下文注入**：生成前通过 `getChatMessages` 获取最近 N 条对话历史，格式化后作为上下文传入 `ordered_prompts`
- 生成帖子时，将对应板块的提示词作为 system prompt，对话历史作为上下文，要求 AI 以 JSON 格式返回帖子数据
- 生成评论时，将帖子内容、已有评论和对话历史作为上下文
- 支持 `json_schema` 强制结构化输出，确保返回格式正确
- 生成过程使用 `should_silence: true` 避免影响酒馆 UI

### 4. 论坛界面 (`components/`)

- **ForumHeader**: 显示可编辑的论坛名称，使用 `contenteditable` 或点击编辑
- **ForumTabs**: 两个板块标签，切换时筛选帖子列表
- **PostList**: 帖子列表，支持「AI 生成新帖」按钮
- **PostItem**: 帖子卡片，显示标题、玩家ID、时间、评论数
- **PostDetail**: 点击帖子进入详情，显示正文和评论区
- **CommentItem**: 单条评论展示
- **PostEditor**: 用户手动发帖/评论的编辑器
- **SettingsPanel**: 设置面板（齿轮图标打开），可编辑：
  - 论坛名称
  - API 模型名
  - 两个板块各自的系统提示词
  - 是否将论坛内容注入酒馆对话（开关）
  - 注入帖子数量上限
  - 生成论坛时读取的对话历史条数

### 5. 双向上下文注入 (`contextInjector.ts`)

#### 论坛 → 酒馆对话

- 使用 `injectPrompts` 持续将论坛帖子注入到酒馆 AI 请求中
- 注入内容格式：
  ```
  [论坛名称 - 当前板块]
  帖子1: 【标题】by 玩家ID
  内容摘要...
  评论: 玩家B: xxx | 玩家C: xxx
  
  帖子2: ...
  ```
- 帖子更新时调用 `uninjectPrompts` 移除旧注入，再用 `injectPrompts` 注入新内容
- 监听 `tavern_events.GENERATION_AFTER_COMMANDS` 确保注入始终最新
- 监听 `tavern_events.CHAT_CHANGED` 在新聊天中重新注入

#### 酒馆对话 → 论坛

- 在 `aiGenerator.ts` 中，生成帖子/评论前获取最近 N 条对话历史
- 将对话历史格式化为摘要文本，作为 `generateRaw` 的 `ordered_prompts` 中的上下文 prompt
- 格式示例：
  ```
  以下是当前剧情概要（最近的对话）：
  用户: xxx
  AI: xxx
  ...
  请根据以上剧情上下文生成论坛帖子
  ```

### 5. 提示词设计

#### 现实讨论板块提示词模板（默认值）

```
你是一个现实世界的网络论坛用户。请根据以下上下文生成论坛帖子和评论。
要求：
- 使用自然的网络用语风格
- 帖子主题围绕现实生活、社会话题、日常讨论
- 每个用户有独特的用户名和发言风格
- 评论应该有不同观点的碰撞
- 帖子内容应当与当前剧情有一定关联性
```

#### 游戏中论坛板块提示词模板（默认值）

```
你是一个网游世界中的玩家。请根据以下游戏上下文生成游戏内论坛的帖子和评论。
要求：
- 使用游戏玩家术语和黑话
- 帖子主题围绕游戏攻略、装备讨论、副本组队、PK恩怨等
- 每个用户有独特的游戏ID和发言风格
- 评论应该体现不同等级、职业玩家的视角差异
- 适当提及游戏中的事件、版本更新、活动等
- 帖子内容应当与当前游戏剧情/事件有关联
```

1. **创建项目结构**: 在 `src/网游论坛/` 下创建 `index.ts` 脚本入口

8. **实现 App.vue**: 悬浮窗容器、布局框架
9. **实现各组件**: 逐一实现 ForumHeader → ForumTabs → PostList → PostItem → PostDetail → CommentItem → PostEditor → SettingsPanel
10. **整合测试**: 确保双向注入和所有功能正常工作

## 部署与配置指南（面向零基础用户）

### 方式一：本地实时开发（推荐先用这个测试）

1. **安装 Node.js**：去 https://nodejs.org 下载安装 LTS 版本
2. **安装 pnpm**：打开终端运行 `npm install -g pnpm`
3. **安装项目依赖**：在项目根目录运行 `pnpm install`
4. **启动实时监听**：运行 `pnpm watch`，这会：
   - 编译 `src/网游论坛/index.ts` 并输出到 `dist/网游论坛/index.js`
   - 启动 socket.io 热重载服务（端口 6621）
5. **在酒馆中导入脚本**：
   - 打开酒馆 → 酒馆助手脚本库
   - 导入 `初始模板/脚本/导入到酒馆中/脚本-实时修改.json`
   - **修改其中内容**：将 `import 'http://localhost:5500/dist/你的脚本打包路径/index.js'` 改为 `import 'http://localhost:6621/dist/网游论坛/index.js'`
   - 启用酒馆助手的「实时监听-允许监听」开关
6. 之后修改代码，酒馆中会自动热重载

### 方式二：上传 GitHub + jsdelivr CDN（发布给他人使用）

这是你看到别人 `https://testingcf.jsdelivr.net/gh/...` 链接的原理：

1. **将项目推送到 GitHub 仓库**（如果还没有的话）
2. 运行 `pnpm build`（生产模式打包），代码会编译到 `dist/网游论坛/index.js`
3. 将代码推送到 GitHub 后，jsdelivr 会自动提供 CDN 访问链接
4. **链接格式**：`https://testingcf.jsdelivr.net/gh/{GitHub用户名}/{仓库名}/dist/网游论坛/index.js`
   - 例如：`https://testingcf.jsdelivr.net/gh/myuser/my-tavern-scripts/dist/网游论坛/index.js`
5. **在酒馆中使用**：在酒馆助手脚本库中新建脚本，内容填写：
   ```
   import 'https://testingcf.jsdelivr.net/gh/myuser/my-tavern-scripts/dist/网游论坛/index.js'
   ```
6. **自动更新**：项目已配置 GitHub Actions 工作流，推送代码后会自动打包并更新版本号，jsdelivr 会自动刷新缓存

### 方式三：本地静态文件（不用 GitHub）

1. 运行 `pnpm build` 打包
2. 用任意方式让 `dist/网游论坛/index.js` 能通过网络访问（如本地 HTTP 服务器）
3. 在酒馆脚本中 `import 'http://你的服务器地址/dist/网游论坛/index.js'`

### 酒馆中的具体配置步骤

1. 打开酒馆 → 点击顶部「Extensions」→ 找到「酒馆助手」
2. 在脚本库中点击「+」新建脚本
3. 脚本内容填写对应的 import 语句（见上方各方式）
4. 启用脚本
5. 酒馆助手脚本库中会出现论坛按钮，点击即可打开/关闭论坛悬浮窗
6. 在论坛设置面板（齿轮图标）中配置：
   - 论坛名称
   - AI 模型名（留空则使用酒馆当前模型）
   - 两个板块的提示词
   - 是否注入论坛内容到对话中
   - 注入帖子数量、对话历史条数

## 样式设计

- 优先使用 tailwindcss（iframe 隔离环境允许使用）
- 论坛风格：深色主题，类似传统论坛/贴吧
- 悬浮窗默认尺寸：约 500×600px
- 适配不同宽度，避免横向滚动条
- 不使用 `vh` 单位，使用固定高度 + `overflow-y: auto`
