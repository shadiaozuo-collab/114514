# 修复论坛脚本：CSS注入 + jsDelivr托管

## 问题诊断

### 1. CSS 空白问题
- **根因**：脚本直接挂载到酒馆主页面 `body`，但 `vue-style-loader` 的 CSS 注入代码虽然存在于 bundle 中，却在运行时未能正确执行（可能是 `module-import` 的 CDN 加载时序问题，导致 CSS 注入函数还未就绪就开始渲染）
- **解决方案**：改用 `createScriptIdIframe` 方式挂载。这是项目原本的方式——创建 iframe，Vue 挂载到 iframe 内部的 document，`teleportStyle` 把酒馆的样式同步进来，CSS 注入在 iframe 内独立运行不受主页面干扰

### 2. 拖动问题
- `createScriptIdIframe` 创建的 iframe 内部无法直接操作父页面移动 iframe
- 需要在 iframe 加载后，通过 `window.parent` 访问父页面，在父页面 document 上注册拖动事件

### 3. jsDelivr 托管问题
- 别人的脚本用 `https://testingcf.jsdelivr.net/npm/xxx/+esm` 是因为发布到了 npm
- 我们的方案：把项目推到 GitHub，然后用 jsDelivr 的 GitHub 模式：`https://cdn.jsdelivr.net/gh/用户名/仓库名@分支/dist/额外API悬浮窗/index.js`
- 用户脚本只需一行 `import 'https://cdn.jsdelivr.net/gh/...'` 即可

### 4. index.js.map 文件
- 这是 source map，用于调试时定位源码位置，不影响运行
- jsDelivr 托管时不需要上传，可以删除

## 实施步骤

### Step 1: 改回 `createScriptIdIframe` 挂载方式
修改 `悬浮窗.ts`：
- 使用 `createScriptIdIframe()` 创建 iframe
- 用 `teleportStyle()` 同步酒馆样式到 iframe
- Vue 挂载到 iframe 的 `contentDocument.body`
- 在父页面用 jQuery 事件委托处理拖动（mousedown 在 iframe header 上，mousemove/mouseup 在父 document 上）
- 保留 💬 悬浮按钮（不走 `appendInexistentScriptButtons`，自己创建）

### Step 2: 修复拖动逻辑
- iframe 内的 header mousedown 事件 → 通过 `window.parent.$(document)` 在父页面注册 mousemove/mouseup
- 或者在 iframe load 后，直接在父页面的 `$container`（iframe jQuery 对象）上注册事件委托

### Step 3: 保持 webpack externals 的 CDN 模式
- 不需要 `@standalone`，用项目原有的 CDN externals
- pinia、klona、dedent 等走 `https://testingcf.jsdelivr.net/npm/xxx/+esm`
- Vue、jQuery、toastr 等走 `var` 全局变量（酒馆已加载）

### Step 4: 构建并验证
- `pnpm build`
- 验证 dist 产物包含正确的 CSS 注入代码
- 验证 index.js 是 ES module 格式（有 `import ... from 'https://...'`）

### Step 5: 提供 jsDelivr 托管说明
- 指导用户把项目推到 GitHub
- 使用 jsDelivr 的 GitHub 模式生成 import URL
- 最终用户脚本只需：`import 'https://cdn.jsdelivr.net/gh/用户名/仓库名/dist/额外API悬浮窗/index.js'`
